# Task: KB-140 - Fix CLI build-exe node-pty native asset packaging

**Created:** 2026-03-30
**Size:** M

## Review Level: 3 (Full)

**Assessment:** This task changes the published CLI's standalone runtime/build pipeline, touches native module loading, and may affect release packaging across platforms. The failure is tightly scoped, but the fix must be validated end-to-end because small mistakes can break `--help`, `task list`, dashboard startup, or binary release artifacts.
**Score:** 6/8 — Blast radius: 2, Pattern novelty: 2, Security: 1, Reversibility: 1

## Mission

Restore the standalone `@dustinbyrne/kb` CLI artifacts generated by `pnpm build` and `pnpm --filter @dustinbyrne/kb build:exe`. Right now the CLI bundle and Bun-compiled binary eagerly pull in dashboard terminal code, and `node-pty` then tries to resolve `prebuilds/<platform-arch>/pty.node` from the bundled runtime (`/$bunfs/root/kb`) even though `packages/cli/build.ts` only stages `dist/client/`. Fix the runtime/loading and packaging path so non-dashboard commands no longer crash, dashboard binaries can find the required native sidecars/helpers from a deterministic filesystem payload next to `dist/`, and isolated build output can not only start the dashboard but also successfully create a real PTY session via `/api/terminal/sessions`.
